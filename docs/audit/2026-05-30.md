## Security review — the new `diffgeo.cyr` functions

### Memory safety — PASS
- `sectional_curvature`, `weyl_tensor`, `geodesic_deviation` are bounded tensor
  contractions (loops over `0..dim`, access via `_dg_idx2/4` / `riemann_get`).
  `weyl_tensor` allocates `dim⁴` (zeroed) — the same size as the Riemann array it
  consumes, so no size is independently attacker-controlled (the dim was already
  validated when `riemann_tensor`, capped at `dim ≤ 16`, built that input).
- `parallel_transport` allocates fixed `dim`-sized RK4 work arrays once per call
  (reused across steps); `geodesic_deviation` allocates one `dim` vector.
- `wedge_2_1` / `wedge_3_1` are 4D-specific (fixed 32 / 8-byte allocations, like
  `hodge_star_2form_4d`); they read the 6-/4-component reduced-basis layout — a
  documented 4D contract on the caller.

### Division by zero — PASS
- `sectional_curvature` `num/denom`: **guarded** — returns 0 when the plane is
  degenerate (`|⟨u,u⟩⟨v,v⟩ − ⟨u,v⟩²| < EPSILON_F64`). Pinned by the
  degenerate-plane test.
- `weyl_tensor` `1/(n−2)` and `R/((n−1)(n−2))`: **guarded** by the early
  `if (dim < 3) return` (so `n−2 ≥ 1`). Pinned by the `n ≤ 2` → zero behaviour.
- `parallel_transport` `inc/6`: divisor is the constant 6.
- `geodesic_deviation` and the wedges perform no division.

### Surface — PASS
No new shell/exec/FFI/libc/syscall surface (pure in-memory tensor math).
